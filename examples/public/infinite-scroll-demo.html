O(delta) virtual list rendering with islice combinator

How Virtual List Works

This demo shows true O(delta) virtual list - only entering/exiting items cause DOM changes:

  • 10,000 items in memory, but only ~10-15 rendered at once
  • Scroll events produce deltas for items entering/exiting viewport
  • islice combinator computes window changes efficiently
  • No virtual DOM diffing - deltas flow directly to DOM operations

Watch the DOM Operations counter - each scroll only adds/removes a few items, not 10!
